# Баг-репорты: sberbusiness.live

**Дата тестирования:** июль 2025
**Окружение:** Chrome (последняя версия), macOS, разрешение 1440×900
**Найдено дефектов:** 3

---

## BUG-01

| Поле | Значение |
|---|---|
| **ID** | BUG-01 |
| **Заголовок** | Поиск по сайту не завершается (зависает) при вводе спецсимвола `$` |
| **Критичность (Severity)** | Major |
| **Приоритет (Priority)** | High |
| **Статус** | Open |
| **Компонент** | Поиск по сайту (модальное окно поиска в шапке) |
| **Тестовая среда** | Chrome (последняя версия), macOS, 1440×900 |
| **Предусловия** | Открыт сайт [sberbusiness.live](https://sberbusiness.live), пользователь не авторизован |
| **Шаги воспроизведения** | 1. Нажать на иконку поиска (лупа) в шапке сайта <br> 2. Ввести в поле поиска символ `$` <br> 3. Дождаться результата поиска |
| **Ожидаемый результат** | Поиск завершается в течение 3–5 секунд. Отображается сообщение «Ничего не найдено» или пустой список результатов |
| **Фактический результат** | Отображается индикатор загрузки «Ищем подходящие публикации». Поиск не завершается более 1 минуты, результат или сообщение об ошибке не выводится |
| **Воспроизводимость** | Стабильно (3 из 3 попыток) |
| **Вложения** | `attachments/bug-01-search-loading.png` — скриншот зависшего индикатора загрузки |
| **Примечание** | Вероятно, спецсимвол не экранируется при формировании поискового запроса. Рекомендуется проверить обработку и других спецсимволов (`%`, `&`, `\`, `"` и т.д.) |
| **Связанный тест-кейс** | [TC-02](test-cases.md#tc-02-поиск-с-вводом-спецсимвола) |

## BUG-02

| Поле | Значение |
|---|---|
| **ID** | BUG-02 |
| **Заголовок** | Отсутствует сообщение об ошибке при отправке формы подписки с пустым полем email |
| **Критичность (Severity)** | Minor |
| **Приоритет (Priority)** | Low |
| **Статус** | Open |
| **Компонент** | Форма подписки на рассылку (футер) |
| **Тестовая среда** | Chrome (последняя версия), macOS, 1440×900 |
| **Предусловия** | Открыт сайт [sberbusiness.live](https://sberbusiness.live), страница прокручена до футера, поле email пустое |
| **Шаги воспроизведения** | 1. Найти форму подписки на рассылку в футере <br> 2. Убедиться, что поле email пустое <br> 3. Нажать кнопку «Подписаться» |
| **Ожидаемый результат** | Отображается сообщение об ошибке вида «Поле обязательно для заполнения». Пользователь понимает, что нужно ввести email |
| **Фактический результат** | Ничего не происходит: сообщение об ошибке не появляется, визуальной реакции на нажатие кнопки нет, форма не отправляется |
| **Воспроизводимость** | Стабильно (3 из 3 попыток) |
| **Вложения** | `attachments/bug-02-empty-field.png` — скриншот формы с пустым полем после нажатия кнопки |
| **Примечание** | Форма фактически защищена от отправки пустого значения, но отсутствие обратной связи ухудшает UX — пользователь не понимает, почему кнопка «не работает» |
| **Связанный тест-кейс** | [TC-04](test-cases.md#tc-04-подписка-с-пустым-полем-email) |

## BUG-03

| Поле | Значение |
|---|---|
| **ID** | BUG-03 |
| **Заголовок** | Форма подписки возвращает необработанную ошибку «Что-то пошло не так!» при отправке валидного email |
| **Критичность (Severity)** | Major |
| **Приоритет (Priority)** | High |
| **Статус** | Open |
| **Компонент** | Форма подписки на рассылку (футер) |
| **Тестовая среда** | Chrome (последняя версия), macOS, 1440×900 |
| **Предусловия** | Открыт сайт [sberbusiness.live](https://sberbusiness.live), страница прокручена до футера |
| **Шаги воспроизведения** | 1. Найти форму подписки на рассылку в футере <br> 2. Ввести в поле email валидный адрес формата `user@domain.ru` <br> 3. Нажать кнопку «Подписаться» |
| **Ожидаемый результат** | Отображается подтверждение успешной подписки либо сообщение о необходимости подтвердить email в письме |
| **Фактический результат** | Появляется модальное окно с текстом «Что-то пошло не так! Попробуйте ещё раз, пожалуйста». Подписка не оформляется, текст ошибки не содержит информации о причине проблемы |
| **Воспроизводимость** | Стабильно (3 из 3 попыток) |
| **Вложения** | `attachments/bug-03-error-modal.png` — скриншот модального окна с ошибкой |
| **Примечание** | Основной пользовательский сценарий (подписка на рассылку) полностью не работает. Рекомендуется проверить ответ сервера в DevTools → Network при отправке формы — вероятно, бэкенд возвращает 4xx/5xx, а фронтенд показывает универсальную заглушку |
| **Связанный тест-кейс** | [TC-05](test-cases.md#tc-05-подписка-с-валидным-email) |
