# Проект: Тестирование сервиса аналитики видеоконтента

## Описание

**Объект тестирования:** веб-сервис аналитики видеоконтента и статистики аккаунтов в социальных сетях *(название скрыто по договорённости с командой разработки)*.

**Контекст:** команда разработки попросила провести функциональное тестирование веб-приложения перед первым публичным релизом.

**Цель:** проверить ключевые пользовательские сценарии — добавление аккаунта социальной сети для отслеживания и обработку видеоконтента.

---

## Что было сделано

- Проведено ручное функциональное тестирование сценариев добавления аккаунта и обработки видео
- Выявлена критичная проблема с транскрибацией длинных разговорных роликов
- Найден визуальный дефект в окне «Добавить аккаунт»
- Оформлен баг-репорт, результаты переданы команде разработки

## Виды тестирования

- Функциональное тестирование
- Тестирование пользовательского интерфейса (UI)

## Инструменты

- Ручное тестирование в браузере
- Chrome DevTools

---

## Результаты

Главный итог тестирования оказался куда весомее оформленного ниже визуального дефекта: сервис на этом этапе не справлялся с транскрибацией длинных разговорных роликов — обработка завершалась ошибкой «не удалось». При этом именно аналитика такого контента была ядром продукта. По результатам тестирования команда доработала функциональность до публичного запуска.

Так что баг-репорт ниже — самая незначительная из находок этого проекта :)

## Баг-репорт

| Поле | Значение |
|---|---|
| **ID** | BUG-01 |
| **Заголовок** | В окне «Добавить аккаунт» после поиска пользователя по нику элементы интерфейса отображаются с нарушением вёрстки |
| **Критичность (Severity)** | Незначительная (визуальный дефект) |
| **Приоритет (Priority)** | Низкий |
| **Статус** | Открыт |
| **Компонент** | Раздел «Аккаунты», модальное окно «Добавить аккаунт» |
| **Тестовая среда** | ОС: macOS Ventura 13.6.4 <br> Устройство: MacBook Pro 15-inch, 2018 (Intel Core i7 2.2 GHz, 16 ГБ DDR4) <br> Браузер: Google Chrome 148.0.7778.168 (x86_64) |
| **Шаги воспроизведения** | 1. Авторизоваться в веб-приложении сервиса <br> 2. Перейти в раздел «Аккаунты» <br> 3. Нажать кнопку «Добавить аккаунт» <br> 4. Ввести в поле поиска ник пользователя и нажать кнопку «Найти» |
| **Ожидаемый результат** | В окне «Добавить аккаунт» корректно отображается карточка найденного пользователя: аватар, ник, описание и число подписчиков не перекрывают друг друга, элементы управления (выбор группы, кнопки «Назад» и «Подтвердить и отслеживать») свёрстаны без смещений |
| **Фактический результат** | Элементы интерфейса отображаются с визуальными артефактами: нарушена вёрстка, элементы перекрывают друг друга, компоненты окна рендерятся некорректно |
